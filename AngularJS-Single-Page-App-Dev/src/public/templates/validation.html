Validation

Form validation is built into Angular as well. It leverages existing HTML5 markup, along with Angular's built in directives to validate form elements, and associate validation results to the form element by placing a property on the $scope at whatever the form's name is. (I realize I stumbled over the wording there, but it's very simple, see the example)

Another thing that validation does in Angular is it will add CSS classes to elements based on their validation state. It adds these elements to both the form and the inputs on the form. This allows you to apply styling to specific items based on their state(s). Items can, of course, be any combination of ng-pristine/ng-dirty and ng-valid/ng-invalid.

Validation CSS classes

  • ng-pristine - for forms and inputs that are in their original, loaded state.
  • ng-dirty - for forms and inputs that have been altered at all.
  • ng-valid - for valid forms and inputs.
  • ng-invalid - for invalid forms and inputs.
